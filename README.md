# 🌱 AMRITKRISHI 2.0

**AMRITKRISHI 2.0** is an AI-powered Smart Farming Ecosystem web application designed to help, educate, and empower Indian farmers. It features a complete end-to-end full-stack architecture built with a modern React Frontend (Vite + TailwindCSS) and a robust Python Backend (FastAPI + Machine Learning).

![AmritKrishi Dashboard Preview](https://i.imgur.com/your-image-url-here.png)

## 🌟 Key Features

- 🌾 **AI Crop Recommendation**: Get precise recommendations on what to grow based on Soil NPK values, pH, temperature, and humidity configurations.
- 📷 **Plant Disease Detection**: Upload photos of infected crop leaves via an interactive drag-and-drop UI to receive instant CNN-powered AI diagnosis and treatment recommendations.
- 🌥️ **Smart Weather Intelligence**: Look up a 5-day weather forecast, micro-climate insights, and dynamic, weather-based farming suggestions (like the optimal date for crop sowing).
- 💬 **Multilingual AI Chatbot**: An interactive WhatsApp-style intelligent Chatbot capable of advising farmers in clear, conversational English, Hindi, and Tamil!
- 📈 **Market Prices**: Live Mandi price boards and analytics charts for major crops.
- 🏛️ **Government Schemes Directory**: A searchable interface to discover and check your eligibility for farming subsidies and insurance plans.

## 🛠️ Tech Stack

**Frontend:**
- **Framework**: React 18, Vite
- **Styling**: Tailwind CSS, Lucide React (Icons)
- **Routing**: React Router v6
- **Data Visualization**: Recharts (for Dashboard & Market analytics)
- **Internationalization**: i18next (English, Tamil, Hindi)

**Backend:**
- **Framework**: Python, FastAPI
- **Machine Learning**: TensorFlow (MobileNetV2 CNN), Scikit-Learn (RandomForest), Joblib
- **Data Fetching**: HTTPX (Async API requests)
- **AI Integration**: Anthropic Claude API

## 🚀 Running the Project Locally

### 1. Clone the Repository
```bash
git clone https://github.com/Archit1906/AI-CROP-RECOMD.git
cd AI-CROP-RECOMD
```

### 2. Backend Setup (FastAPI + ML)
Open a terminal in the root folder.
```bash
cd backend
python -m venv venv
# On Windows:
venv\Scripts\activate
# On Mac/Linux:
source venv/bin/activate

# Install all backend requirements
pip install -r requirements.txt

# Run the mock training scripts to generate the dummy models necessary to boot the app
python ml/train_crop.py
python ml/train_disease.py

# Create a `.env` file containing your API Keys
# Check backend/.env for required format (Anthropic, OpenWeather, etc.)

# Start the Python Development Server
python -m uvicorn main:app --reload --port 8000
```
Your backend will be available at `http://localhost:8000`. You can test the endpoints interactively via the autogenerated Swagger Documentation at `http://localhost:8000/docs`.

### 3. Frontend Setup (React + Vite)
Open a new, separate terminal tab in the root folder.
```bash
cd frontend

# Install Node dependencies
npm install

# Create a `.env` file containing your backend integration path
# VITE_API_URL=http://localhost:8000/api

# Start the Vite Development Server
npm run dev
```
Navigate to `http://localhost:5173` to explore the complete frontend User Interface!

## 🌐 Deployment Configuration

This project is configured out of the box for modern, easy cloud deployment:

### Deploying the Frontend (Vercel)
1. Push this repository to GitHub.
2. Sign in to [Vercel](https://vercel.com/) and create a "New Project".
3. Import your GitHub repository. Vercel will automatically detect `frontend/` as a Vite Application if you set the Framework Preset and specify `frontend` as the Root Directory.
4. Set the `VITE_API_URL` environment variable in Vercel to your deployed backend's URL.

### Deploying the Backend (Render)
A `render.yaml` configuration is provided for 1-click Render deployments.
1. Create an account on [Render](https://render.com/).
2. Select **New** > **Blueprint**.
3. Connect your repository. Render will utilize the blueprint to provision a Python Web Service automatically running your FastAPI backend.
4. Don't forget to configure your production API Key variables via the Render Dashboard.
